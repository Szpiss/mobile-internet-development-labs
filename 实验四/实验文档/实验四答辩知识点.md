# 实验四答辩知识点：Html5 设计展邀请函

## 一、项目一句话介绍

本实验使用 HTML5、CSS3、JavaScript 和 Swiper 插件完成了一个移动端 H5 邀请函页面，页面包含三个竖向滑动页面，分别展示活动封面、活动介绍和创意作品展示，并为指定元素添加了进入动画。

答辩时可以这样说：

> 这个实验主要练习 Swiper 插件在移动端 H5 页面中的使用。我完成了一个三页竖向滑动的 Html5 设计展邀请函，第一页标题使用 swing 动画，第二页标题使用 bounceIn 动画，第三页 pic1 图片使用 zoomIn 动画，同时实现了分页器、滚轮切换和本地备用切换逻辑。

## 四、HTML 相关知识点

### 1. HTML5 基本结构

本实验使用了标准 HTML5 文档结构：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
  ...
</body>
</html>
```

可能问题：`<!DOCTYPE html>` 有什么作用？

回答：

> 它用于声明当前页面使用 HTML5 标准，帮助浏览器以标准模式解析页面，避免进入兼容模式导致样式表现不一致。

可能问题：`meta viewport` 有什么作用？

回答：

> `viewport` 用来控制移动端页面的视口宽度和缩放比例。这里设置 `width=device-width`，让页面宽度等于设备宽度；设置 `initial-scale=1.0`，让页面按 1 倍比例显示，适合移动端 H5 页面。

### 2. 语义化标签

本实验使用 `main`、`section`、`article`、`h1`、`h2`、`dl`、`dt`、`dd` 等标签。

可能问题：为什么用 `article` 表示每一页？

回答：

> 每一页邀请函内容相对独立，所以用 `article` 表示一个独立内容块，同时它也是 Swiper 的一个 slide。这样结构更清楚，后续维护时也更容易定位每一页。

### 3. 图片的 `alt` 属性

第三页图片使用：

```html
<img class="pic1 ani" src="img/pic1.png" alt="设计展作品拼贴图">
```

可能问题：`alt` 有什么作用？

回答：

> `alt` 是图片的替代文本。图片加载失败时可以显示文字，同时也有利于无障碍访问和搜索引擎理解图片内容。

## 五、Swiper 相关知识点

### 1. Swiper 是什么

Swiper 是一个常用于移动端的触摸滑动插件，可以快速实现轮播图、全屏滑动页、图片画廊等交互效果。

答辩时可以这样说：

> 本实验用 Swiper 主要是为了完成移动端 H5 邀请函中常见的整屏滑动效果，它比自己手写触摸事件更稳定，也更符合实际开发习惯。

### 2. Swiper 基本结构

Swiper 的核心结构是：

```html
<section class="swiper invite-swiper">
  <div class="swiper-wrapper">
    <article class="swiper-slide">第一页</article>
    <article class="swiper-slide">第二页</article>
    <article class="swiper-slide">第三页</article>
  </div>
  <div class="swiper-pagination"></div>
</section>
```

三层结构含义：

- `.swiper`：最外层容器。
- `.swiper-wrapper`：包裹全部滑动页。
- `.swiper-slide`：每一个可以滑动切换的页面。

可能问题：为什么必须有 `.swiper-wrapper`？

回答：

> Swiper 会控制 wrapper 的位移来实现滑动切换，slide 都需要放在 wrapper 里面，插件才能计算每一页的位置和切换效果。

### 3. Swiper 初始化

本实验核心初始化代码：

```javascript
var swiper = new Swiper(".invite-swiper", {
  direction: "vertical",
  loop: false,
  initialSlide: initialSlide,
  speed: 620,
  mousewheel: true,
  pagination: {
    el: ".swiper-pagination",
    clickable: true
  }
});
```

配置说明：

- `direction: "vertical"`：竖向滑动。
- `loop: false`：不循环播放，符合三页邀请函顺序展示。
- `initialSlide`：指定初始显示第几页，截图时也方便定位。
- `speed: 620`：滑动动画持续时间，单位毫秒。
- `mousewheel: true`：允许桌面浏览器使用鼠标滚轮切换。
- `pagination.el`：指定分页器元素。
- `pagination.clickable: true`：允许点击分页器切换页面。

可能问题：为什么设置 `loop: false`？

回答：

> 因为邀请函是从封面到介绍再到作品展示的线性阅读顺序，不循环更符合内容逻辑。三页看完后停在最后一页，用户更容易理解页面已经结束。

可能问题：`direction: "vertical"` 改成 `horizontal` 会怎样？

回答：

> 页面会从竖向切换变为横向切换。实验要求是竖直方向滑动，所以这里使用 `vertical`。

## 六、动画实现知识点

### 1. 动画属性设计

页面元素通过属性标记动画：

```html
<h1 class="title ani"
    swiper-animate-effect="swing"
    swiper-animate-duration="0.9s"
    swiper-animate-delay="0.2s">
  Html5设计展
</h1>
```

含义：

- `ani`：标记该元素需要执行动画。
- `swiper-animate-effect`：动画名称。
- `swiper-animate-duration`：动画持续时间。
- `swiper-animate-delay`：动画延迟时间。

可能问题：为什么要加 `ani` 类？

回答：

> `ani` 相当于一个统一标记。JS 可以通过 `.ani` 找到当前页面里所有需要动画的元素，然后读取它们的动画属性并触发动画。

### 2. CSS 关键帧动画

本实验定义了 `swing`、`bounceIn`、`zoomIn` 等关键帧动画。

`swing`：模拟标题左右摆动。

`bounceIn`：先缩小进入，再略微放大回弹。

`zoomIn`：从小到大放大出现。

可能问题：CSS 动画的核心组成是什么？

回答：

> CSS 动画主要由 `@keyframes` 和 `animation` 相关属性组成。`@keyframes` 定义动画过程，`animation-name`、`animation-duration`、`animation-delay` 等属性控制动画名称、时间和延迟。

### 3. 为什么滑动后要重新触发动画

`main.js` 中有两个核心函数：

```javascript
resetAnimations(document);
runAnimations(this.slides[this.activeIndex]);
```

作用：

- `resetAnimations`：先清除旧动画类和内联动画属性。
- `runAnimations`：给当前页元素重新添加动画类。

可能问题：为什么要先清除再添加动画类？

回答：

> 浏览器对同一个元素已经执行过的动画不会每次都自动重播。先移除动画类和样式，再重新添加，可以让动画在每次进入当前页面时重新播放。

可能问题：`void element.offsetWidth` 是做什么的？

回答：

> 这行代码用于强制浏览器重新计算元素布局，让前面移除动画类的操作真正生效。之后再添加动画类时，动画就能被重新触发。

## 七、CSS 布局知识点

### 1. 移动端画布

页面最外层使用 `.phone-stage`：

```css
.phone-stage {
  width: min(100vw, 430px);
  height: min(100vh, 760px);
  min-height: 560px;
  overflow: hidden;
}
```

可能问题：为什么设置最大宽度 430px？

回答：

> 邀请函是移动端 H5 页面，最大宽度限制在接近手机屏幕的尺寸，可以让桌面浏览器预览时也保持手机页面比例，不会被拉得太宽。

### 2. `overflow: hidden`

可能问题：为什么容器要设置 `overflow: hidden`？

回答：

> Swiper 页面切换时，其他 slide 会在容器外面移动。设置 `overflow: hidden` 可以隐藏超出当前手机画布范围的内容，只显示当前页面。

### 3. 背景图和 `object-fit`

```css
.slide-bg {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

可能问题：`object-fit: cover` 有什么作用？

回答：

> 它可以让图片保持比例填满容器，必要时裁剪多余部分，避免图片被拉伸变形。适合做整页背景图。

### 4. 绝对定位

背景图使用 `position: absolute`，内容区域使用 `position: relative`。

可能问题：为什么背景图用绝对定位？

回答：

> 背景图需要铺满当前 slide 并位于内容下方，所以用绝对定位覆盖整个页面；文字内容设置相对定位并提高层级，保证显示在背景图上面。

## 八、JavaScript 相关知识点

### 1. IIFE 立即执行函数

`main.js` 使用：

```javascript
(function () {
  ...
})();
```

可能问题：为什么这样包一层函数？

回答：

> 这是立即执行函数，可以避免变量直接暴露到全局作用域，减少与其他脚本命名冲突的可能。

### 2. `querySelectorAll`

```javascript
container.querySelectorAll(".ani")
```

可能问题：这句代码有什么作用？

回答：

> 它会查找容器内所有类名包含 `ani` 的元素，这些元素就是需要执行动画的元素。

### 3. `classList.add` 和 `classList.remove`

可能问题：为什么用 `classList`？

回答：

> `classList` 可以方便地添加和删除 CSS 类。这里通过添加 `effect-swing`、`effect-bounceIn`、`effect-zoomIn` 等类来触发对应 CSS 动画。

### 4. `URLSearchParams`

```javascript
new URLSearchParams(window.location.search).get("slide")
```

可能问题：这里为什么读取 URL 参数？

回答：

> 这是为了方便生成截图和调试。比如访问 `index.html?slide=2` 可以直接打开第三页，便于截取每一页效果图。

### 5. 备用滑动逻辑

代码中写了 `fallbackSwiper()`。

可能问题：为什么要写备用逻辑？

回答：

> 页面依赖 CDN 加载 Swiper。如果网络异常导致 Swiper 没有加载成功，备用逻辑可以用基础 JavaScript 实现简单的切换，保证页面仍然能基本展示和切换。

## 十、老师可能追问的问题与简答

### 1. 你这个实验最核心的代码是哪一部分？

回答：

> 最核心的是 Swiper 的 HTML 结构和 `new Swiper()` 初始化配置。HTML 中三个 `.swiper-slide` 决定页面内容，JS 中 `direction: "vertical"` 决定竖向滑动，pagination 和 mousewheel 决定交互方式。

### 2. Swiper 的三个核心类名是什么？

回答：

> 分别是 `.swiper`、`.swiper-wrapper` 和 `.swiper-slide`。外层容器负责初始化，wrapper 包裹所有页面，slide 表示单个滑动页。

### 3. 为什么这个页面适合做成 H5 竖向滑动？

回答：

> 邀请函内容通常是按顺序展示的，比如封面、活动介绍、作品展示。竖向滑动符合移动端阅读习惯，也能让每一页内容聚焦在一个主题上。

### 4. 如果要增加第四页，你会怎么做？

回答：

> 在 `.swiper-wrapper` 里面新增一个 `.swiper-slide`，写入第四页内容。如果有动画，就给元素添加 `.ani` 和对应的 `swiper-animate-effect` 属性。Swiper 会自动识别新增 slide。

### 5. 如果要改成自动播放，应该怎么做？

回答：

> 可以在 Swiper 配置中添加 `autoplay`，比如 `autoplay: { delay: 3000 }`，表示每 3 秒自动切换一页。

### 6. 如果图片显示不出来，可能是什么原因？

回答：

> 常见原因是图片路径写错、图片文件没有放到对应目录、文件名大小写不一致，或者浏览器没有权限访问本地文件。本实验中图片统一放在 `img` 目录，并使用相对路径引用。

### 7. CSS 动画和 JavaScript 动画有什么区别？

回答：

> CSS 动画适合做样式过渡和视觉效果，比如缩放、淡入、摆动；JavaScript 更适合控制动画什么时候触发、触发哪个元素。本实验是用 JavaScript 控制触发时机，用 CSS 定义具体动画效果。

### 8. 你的动画为什么不是一直播放？

回答：

> 邀请函的动画主要用于页面进入时增强表现力，如果一直循环会影响阅读。所以本实验设置为进入当前页时播放一次。

### 9. 为什么要把 CSS 和 JS 分开写？

回答：

> HTML 负责结构，CSS 负责样式，JS 负责交互。分开写可以让代码更清晰，后续修改页面内容、视觉样式或交互逻辑时互不干扰。

### 10. 你在这个实验里遇到的重点或难点是什么？

回答：

> 重点是搭建正确的 Swiper 结构，并让动画在每次进入页面时重新触发。因为 CSS 动画执行过一次后不会自动重复播放，所以我在 JS 中写了重置和重新添加动画类的逻辑。

### 11. 为什么第三页图片叫 pic1？

回答：

> 实验要求中明确提到第三页图片 `pic1` 使用 zoomIn 效果，所以我在第三页使用 `pic1.png` 作为作品展示图，并给它设置 zoomIn 动画。

### 12. 这个项目还有什么可以改进？

回答：

> 可以继续增加背景音乐、报名按钮、自动播放、更多页面动画，或者把活动时间地点改成可配置数据。也可以将 Swiper 文件下载到本地，避免依赖 CDN 网络。
